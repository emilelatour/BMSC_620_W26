R: Working with data using the tidyverse

Author

Emile Latour

Published

January 14, 2026

Where we are

Goal for today (R portion)

We are learning a small, reusable “grammar” for working with data:

  • tidyverse (what it is)
  • tibbles (modern data frames)
  • subsetting (keep rows/columns)
  • pipes (%>%) as “and then”
  • core dplyr verbs: filter(), select(), mutate(), summarize(), group_by()

Why this matters

Next class we start ggplot2 + EDA.

Most plots start with:

  1. keep the rows you care about (filter())
  2. keep the variables you need (select())
  3. create a helper variable (mutate())
  4. plot (ggplot())

The tidyverse

What is the tidyverse?

The tidyverse is a collection of R packages designed to work together:

  • dplyr: data manipulation
  • ggplot2: visualization
  • readxl: import Excel files (we’ll use later)

We are not trying to learn everything in the tidyverse.

Installing vs loading (quick reminder)

Install (once per computer):

install.packages("tidyverse")

Load (every session):

library(tidyverse)

Data frames and tibbles

Data frames vs tibbles

A data frame is a rectangular table:

  • rows = observations
  • columns = variables

A tibble is a modern data frame with nicer printing.

Most tidyverse functions return tibbles.

Subsetting: base R vs dplyr

Subsetting in base R: [rows, cols]

df[rows, cols]

Examples:

df[1:5, ]                 # first 5 rows
df[, c("name", "height")] # two columns

Logical subsetting in base R

Keep rows where a condition is TRUE:

df[df$species == "Human", ]

Read it out loud:

“Give me the rows where species equals Human.”

Pipes

The pipe %>% means “and then”

Pipes help you read code top-to-bottom.

df %>%
  filter(species == "Human") %>%
  select(name, height, mass)

You can always write code without pipes. We use them for readability.

Core dplyr verbs

The 5 verbs we will use a lot

  • filter() keep rows
  • select() keep (or drop) columns
  • mutate() create new columns
  • summarize() collapse to summaries
  • group_by() do summaries by groups

If you learn these five, you can do most things we need in this course.

summarize(): reduce to a summary row

df %>%
  summarize(
    n = n(),
    mean_height = mean(height, na.rm = TRUE),
    mean_mass = mean(mass, na.rm = TRUE)
  )
# A tibble: 1 × 3
      n mean_height mean_mass
  <int>       <dbl>     <dbl>
1    87        175.      97.3

group_by() + summarize(): summaries by group

df %>%

Task 1: Humans only

  3. Show the first 10 rows

Task 2: Conditional summaries

Compute the mean height by sex:

df %>%
  group_by(sex) %>%
  summarize(
    n = n(),
    mean_height = mean(height, na.rm = TRUE),
    .groups = "drop"
  )

Recap

What you should remember

  • tidyverse = a toolkit (we’ll mainly use dplyr + ggplot2 + readxl)
  • tibbles are modern data frames
  • filter() rows, select() columns
  • %>% means “and then”
  • mutate() create columns
  • summarize() + group_by() for conditional summaries
